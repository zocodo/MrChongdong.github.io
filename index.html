    注意：上面搭建步骤中的更换域名不是必须的，对域名没有要求的请忽略

  • 这一步解决对于 hexo 开启服务器无法检测文件更改从而刷新网页的

    • 全局安装 Browsersync
    • 安装结束后执行下面命令，能显示对应版本则表示安装成功
    • 安装 hexo 端 Browsersync 插件
    • 重启 hexo 服务器

  • 将博客的文章分类（如果主题不带menu项（步骤2），请换一个）

    1. 博客文件夹根目录下打开/scaffolds/post.md文件，在tags上添加categories:,文件会变成这样，以后每次执行 hexo n 文件名 都会带有categories项

    2. 打开你使用的主题文件夹，也就是 /themes/xxx，再用文件编辑器打开主题目录里的_config.yml文件，找到menu栏，你会看到类似于下面的那些配置

    3. 然后 cmd 执行hexo n 文件名创建一个新的文章，根据提示路径找到那个文件，编辑器打开后在categories:写上你的分类，例如categories: redux (这个就相当于路径categories/redux/)，也正好匹配了步骤2 中的 Redux 这个分类的路径

      然后在页面点击 Redux 按钮就可以看到设置为categories:redux所有文章，搭建一个带分类的blog就大功告成了

Redux菜鸟入门

一个简单的 redux 项目只需要几步

  1. 创建store

    使用 createStore 方法创建 store ，值得注意的是 store 在 redux 里是唯一的，存储页面所有的 state，而且只允许通过 reducers 来改变它，下面开始讲解创建一个 store 步骤，以及一些遇到的坑

    createStore( reducer , [initialState] , enhancer )

    该方法可以传三个参数

    • reducer 唯一可以改变 store的方法，reducer就是一个函数，每一个reducer只负责更新state的一部分，怎么触发reducer继续往下走

      假如我有两个这样的reducer呢，又不能创建两个store，官方给我们提供了combineReducers方法来把多个reducer组合成一个reducer，使用方法：

      那么我们是怎样触发这个reducer的呢？通过dispatch(action)方法触发，每一个action决定更新state的一部分数据，在这里提一下，action我不喜欢按照官方那样写成一个方法来引入（个人喜欢），如果你喜欢按照官方的方法创建action，请移步到：http://cn.redux.js.org/docs/basics/Actions.html

      当store被创建好以后会提供了以下的方法

      getState() 方法获取 state；
      dispatch(action) 方法更新 state；
      subscribe(listener) 注册监听器；

    • initialState可选, 用于设置 state初始状态

      ​

    • enhancer可选择使用与否 , 中间件( action –> 中间件 ( dispatch()在这里被添加了一些功能 ) –>reducer )，用于增强store.dispatch()的方法，可以利用第三方的插件，好比如redux-thunk

      ​

    • 在这里讲一下中间件是什么? 中间件实质上就是对store.dispatch()进行了改造增强 , 因为reducer的每次更新都需要通过dispatch(action)来触发 , 使用了中间件后 , 每次调用dispatch()时实际调用的都你改造增强后的dispatch()方法

      源码解读推荐查看: http://simplyy.space/article/577a04850c83425c446a0f83

      ​

      上面就已经把创建store的方法介绍完了，先缓一缓，后面讲解怎么把store和组件连接起来….

  2. Provider

    Provider这个模块是作为整个App的容器，在你原有的App Container的基础上再包上一层，它的工作很简单，就是接受Redux的store作为props，并将其声明为context的属性之一，子组件可以在声明了contextTypes之后可以方便的通过this.context.store访问到store。不过我们的组件通常不需要这么做，将store放在context里，是为了给下面的connect用的。

  3. connect连接store和组件

    在你需要用到state的组件注入，注入关键方法connect，你就可以在对应的组件里使用this.props拿到对应state的数据了，也可以通过this.props.dispatch(action)来更新redux的store，最好用到哪个state就传哪个state，避免不必要的性能损耗
